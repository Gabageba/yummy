# Yummy

## Запуск на мобильном устройстве

Проверка приложения в браузере телефона (тот же Wi‑Fi, что и компьютер).

### Требования

- Телефон и компьютер в одной Wi‑Fi сети.
- Запущены оба сервиса: API и фронт.

### Шаги

1. **Запустите API по HTTP** (в каталоге `yummy-api`):
   ```bash
   cd yummy-api
   npm run start:dev:mobile
   ```
   Режим `start:dev:mobile` отключает HTTPS (`DISABLE_SSL=true`), чтобы запросы с телефона не блокировались из‑за самоподписанного сертификата.

2. **Запустите фронт для мобильного** (в каталоге `yummy-web`):
   ```bash
   cd yummy-web
   npm run dev:mobile
   ```
   Скрипт определит локальный IP и подставит его в `VITE_API_URL`. В консоли появится адрес для открытия на телефоне, например:
   ```
   Запуск для телефона: откройте в браузере http://192.168.x.x:3000
   API URL: http://192.168.x.x:8888
   ```

3. **На телефоне** откройте в браузере указанный адрес (например `http://192.168.x.x:3000`).

### Обычный запуск (без мобильного)

- **API:** в `yummy-api` — `npm run start:dev` (с HTTPS по умолчанию).
- **Фронт:** в `yummy-web` — `npm run dev` или `npm start` (обращения к `localhost`).
