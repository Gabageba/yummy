# Yummy -- План улучшения дизайна

## Текущее состояние

Приложение имеет хорошую основу: тёплая оранжевая палитра (`#F7A348`), адаптивная вёрстка с мобильной нижней навигацией, BEM-стиль SCSS и интеграция с Ant Design. Однако есть ряд областей, где дизайн можно значительно улучшить, лучше используя возможности antd и добавив современные UX-паттерны.

---

## 1. Консолидация и расширение темы Ant Design (Высокий приоритет)

**Проблема:** Существуют две параллельные системы токенов -- CSS-переменные в `[_colors.scss](yummy-web/src/styles/variables/_colors.scss)` и JS-объект `COLORS` в `[antdTheme.ts](yummy-web/src/utils/antdTheme.ts)`. Значения дублируются, а конфигурация темы antd минимальна (всего 4 токена + фон Layout). Это приводит к `!important`-переопределениям в `[_antd.scss](yummy-web/src/styles/_antd.scss)`.

**Предложение:**

- Расширить `antdTheme.ts`, добавив больше токенов: `borderRadius`, `colorBgContainer`, `colorTextSecondary`, `colorBorder`, `fontSize`, `fontSizeHeading`*, `marginSM/MD/LG`, `controlHeight` и др.
- Добавить покомпонентные токены для `Card`, `Button`, `Modal`, `Input`, `Tag`, `Menu` -- это устранит большинство SCSS-переопределений и хаков с `!important`.
- Включить генерацию CSS-переменных из antd-токенов (`cssVar: true` в конфигурации темы) для унификации обеих систем.

```typescript
export const antdTheme: ThemeConfig = {
  cssVar: true,
  token: {
    colorPrimary: '#f7a348',
    colorText: '#111827',
    colorLink: '#f7a348',
    colorBgLayout: '#fff',
    borderRadius: 12,
    fontWeightStrong: 500,
    colorSuccess: '#52c41a',
    colorWarning: '#faad14',
    colorError: '#ff4d4f',
  },
  components: {
    Layout: { headerBg: '#fff', bodyBg: '#fff' },
    Card: { actionsBg: 'transparent' },
    Button: { borderRadius: 10 },
    Menu: { itemBg: 'transparent' },
    Modal: { borderRadiusLG: 16 },
  },
};
```

## 2. Исправить сломанную переменную `--color-secondary` (Баг)

**Проблема:** В `[PageLayout/index.scss:42](yummy-web/src/components/core/PageLayout/index.scss)` используется `var(--color-secondary)` для фона меню, но `--color-secondary` закомментирована в `_colors.scss` (строка 4). Меню, вероятно, рендерится без фона или с фолбэком.

**Исправление:** Либо раскомментировать и определить `--color-secondary` в `_colors.scss`, либо заменить использование на `var(--color-bg-secondary)` / `transparent`.

---

## 3. Skeleton-загрузка вместо простого Spin (Высокое влияние)

**Проблема:** Все состояния загрузки используют `<Spin spinning />` -- для загрузки страниц (`[PageLayout/index.tsx:46](yummy-web/src/components/core/PageLayout/index.tsx)`) и списков карточек (`[CardsList/index.tsx:45](yummy-web/src/components/core/CardsList/index.tsx)`). Это создаёт резкий переход -- пустая страница + спиннер.

**Предложение:**

- Использовать `<Card loading />` или `<Skeleton>` для списков карточек -- у компонента `Card` в antd есть встроенный проп `loading`, рендерящий скелетон.
- В `CardsList` рендерить N скелетон-карточек (по количеству `columnsCount`) во время загрузки, вместо оборачивания всего в `<Spin>`.
- Для `PageLayout` использовать `<Skeleton>` блоки, повторяющие структуру страницы (скелетон заголовка + описания, затем скелетон контента).

---

## 4. Улучшение дизайна карточек (Высокое влияние)

**Проблема:** И `DishCard`, и `CollectionCard` используют `CoverIcon` -- статический градиент с фиксированной иконкой. Все карточки выглядят одинаково. Hover-эффект -- только смена цвета рамки.

**Предложения:**

- **Поддержка изображений:** Добавить возможность загрузки обложек для блюд/коллекций; использовать градиент+иконку как фолбэк при отсутствии изображения.
- **Улучшенный hover-эффект:** Добавить `transform: translateY(-2px)` и `box-shadow` при наведении для эффекта "приподнятия":

```scss
.card {
  transition: border-color 0.2s, transform 0.2s, box-shadow 0.2s;
  
  &:hover {
    border-color: var(--color-primary);
    transform: translateY(-2px);
    box-shadow: 0 8px 24px rgba(0, 0, 0, 0.08);
  }
}
```

- **Адаптивная высота обложки:** Сделать высоту `CoverIcon` адаптивной (вместо фиксированных 192px) -- меньше на мобильных.
- **Обрезка текста:** Добавить `ellipsis` к `Typography.Title` и `Typography.Text` в карточках, чтобы они не переполнялись и карточки оставались одинаковой высоты.
- **Стилизация action-иконок:** Добавить hover-эффекты к иконкам действий (плавная смена цвета, масштабирование).

## 5. Унификация системы отступов (Средний приоритет)

**Проблема:** В кодовой базе смешаны три разные системы отступов:

- CSS-переменные: `var(--padding-xl)`, `var(--margin-sm)`
- Antd-токены: `token.margin`, `token.marginXS`
- Захардкоженные значения: `24`, `16` (например, `gutter={[16, 16]}` в `CardsList`, `gap={24}` в `LoginPage`)

**Предложение:** Выбрать одну систему и использовать её последовательно:

- Рекомендуемый подход: antd-токены (`token.`*) в TSX и CSS-переменные (генерируемые из antd через `cssVar: true`) в SCSS.
- Заменить все захардкоженные пиксельные значения на токены или переменные.

---

## 6. Переходы между страницами и микро-анимации (Среднее влияние)

**Проблема:** Нет переходов между маршрутами; контент просто появляется/исчезает.

**Предложения:**

- Добавить fade-in анимацию для контента страницы через CSS или `framer-motion`.
- Добавить анимации появления для списков карточек (поэтапный fade-in для каждой карточки).
- Существующие keyframes `float-`* и `pulse` в `[_global.scss](yummy-web/src/styles/_global.scss)` определены, но могут использоваться шире.

---

## 7. Улучшение пустых состояний (Среднее влияние)

**Проблема:** Компонент `Empty` использует `PRESENTED_IMAGE_SIMPLE` -- базовую иллюстрацию antd. Для приложения о еде это выглядит слишком стандартно.

**Предложение:**

- Создать кастомную иллюстрацию пустого состояния в стиле тёплой оранжевой палитры бренда (например, пустая тарелка, книга рецептов).
- Добавить контекстные сообщения: "Блюд пока нет -- создайте первое!" с CTA-кнопкой, вместо стандартного "Нет данных".

---

## 8. Улучшение мобильной навигации (Средний приоритет)

**Проблема:** Мобильная нижняя навигация (`[MobileHeader](yummy-web/src/components/core/PageLayout/headers/MobileHeader/index.tsx)`) функциональна, но визуально проста -- иконки и текст в сетке `Row/Col` без визуальной обратной связи, кроме смены цвета.

**Предложения:**

- Добавить индикаторную точку или полоску под активным элементом.
- Добавить тонкую анимацию масштабирования при нажатии.
- Рассмотреть использование `Segmented` из antd или кастомного tab bar для более отполированной навигации.
- Добавить `safe-area-inset-bottom` для устройств iOS с домашним индикатором.

---

## 9. Улучшение UX форм в модалках (Среднее влияние)

**Предложения:**

- Добавить счётчики символов для полей имени/описания.
- Добавить `autoFocus` на первое поле при открытии модалки.
- Использовать `Drawer` вместо `Modal` на мобильных устройствах для более нативного опыта (выезд снизу на весь экран).
- Добавить визуальную обратную связь (уведомление / toast) после успешного создания/редактирования.

---

## 10. Улучшение типографической иерархии (Низкий-Средний приоритет)

**Проблема:** Уровни типографики используются (`level={2}`, `level={4}`), но глобальная шкала типографики не настроена в теме antd.

**Предложение:** Определить размеры заголовков, межстрочные интервалы и начертания в токенах темы antd:

```typescript
token: {
  fontSizeHeading2: 28,
  fontSizeHeading3: 22,
  fontSizeHeading4: 18,
  lineHeightHeading2: 1.3,
  // ...
}
```

---

## 11. Диалоги подтверждения для деструктивных действий (UX-баг)

**Проблема:** `DeleteDishAction` и действие удаления в `CollectionCard` вызывают мутации удаления напрямую, без подтверждения.

**Предложение:** Добавить `Modal.confirm` с предупреждением перед удалением:

```typescript
Modal.confirm({
  title: t('confirmDelete'),
  content: t('deleteWarning'),
  okType: 'danger',
  onOk: () => deleteDish(dishId),
});
```

---

## 12. Поддержка тёмной темы (На будущее)

**Проблема:** Существует только светлая тема. CSS-переменные захардкожены на светлые цвета.

**Предложение:** Это более масштабная задача, но можно заложить фундамент:

- Использовать `algorithm: theme.darkAlgorithm` из antd для тёмного варианта.
- Обернуть CSS-переменные в селектор `[data-theme="dark"]` для переопределения цветов.
- Добавить переключатель на странице настроек.
